# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Browser-based agent-based market simulator for studying **bubbles under bounded
rationality with a regulator intervention**. A population of ≤ 200 utility-
maximizing traders participates in a continuous double auction where the
per-period dividend is generated by a **two-state hidden Markov model** (HMM).
The true fundamental value is computable analytically, but not trivially — the
HMM forces agents with weaker cognitive capacity to estimate FV from observed
dividends using bounded-memory filters. A **regulator** observes the rolling
price-to-FV ratio and posts warning messages to the agent bus when the market
ratio sustains above a threshold; agent responses differ by reaction profile
(ignore / discount / panic).

Pure HTML + CSS + vanilla JavaScript. No frameworks, no build step, no
dependencies. Open `index.html` directly in a browser to run.

## Architecture

Pipeline (top to bottom; each layer only talks to the one below):

```
main.js        App state + param-panel wiring + render scheduling.
engine.js      Simulation loop + seeded mulberry32 PRNG.
               ↓
dividend.js    HMMDividend — two-state regime-switching generator with
               expectedRemaining() lookup (for analytical FV).
estimator.js   Naive / Kalman / Analytical FV estimators.
market.js     ─ Order / Trade / OrderBook (price-time priority) / Market.
messaging.js ─ MessageBus (bounded ring buffer + archive) + TrustMatrix
               (Float32Array(N*N) with topK()/meanInbound()).
regulator.js ─ Bubble-threshold monitor; posts regulator-warning
               messages to the bus, may halt the book under 'halt' mode.
agents.js    ─ UtilityAgent with { riskPref, cognitiveType,
               regulatorReaction, biasMode, isLLM, traced } + sampler.
llm.js       ─ LLMAgent scaffold (deterministic stub). Prompt builder
               includes a MARKET ALERTS section populated from the
               regulator; the stub returns a heuristic action in the
               same JSON shape a real Claude/OpenAI call would.
logger.js    ─ Per-agent opt-in traceStore + ring-buffered aggregateStore
               (per-tick wealth/valuation quantiles, action hist, regulator
               state). Sparse events kept in full.
               ↓
replay.js      Build 'view' objects from Market + Logger + Trust state,
               either live (buildLiveView) or at a historical tick
               (buildViewAt). Exposes isTraced(id) so the UI can branch
               between rich trace and aggregate for any given agent.
               ↓
viz.js         HiDPI canvas primitives: axes, line, area, bars,
               fanChart (IQR + envelope), drawHeatmap (N×N).
ui.js          DOM + canvas rendering. Consumes views only.
```

### Invariants

- **No agent class other than `UtilityAgent`.** Variety comes from the
  orthogonal axes on a spec (risk × cognitive × regulator-reaction ×
  bias × LLM-flag). Plan I = algorithmic argmax path; Plan II / III run
  the same agent through the LLM scaffold (`js/llm.js`) with different
  prompt variants.
- **The hidden HMM regime is never exposed to agent code.** Agents only
  observe realised dividends and must filter for the regime themselves
  through their `estimator`.
- **Only the AnalyticalEstimator is authoritative.** The regulator and the
  displayed "reference FV" both read from a single shared
  `refEstimator = new AnalyticalEstimator(hmm)` owned by the Engine.
- **Logging is opt-in per agent.** `agent.traced === true` records a full
  decision trace; otherwise the agent is rolled into the per-tick cohort
  aggregate (quantiles only). Toggle at runtime via the table checkbox.
- **History arrays on `Market` are append-only**; `aggregateStore` is a
  ring buffer. Exact replay inside the window, aggregate-only outside.
- **Randomness flows through the seeded RNG** — two independent streams,
  `makeRNG(seed)` for the engine and `makeRNG(seed ^ 0xA5A5A5A5)` for
  sampling — so endowment edits + rebuild reproduce the same trading
  sequence as a matched run without an edit.
- **UI reads only a view object** from `replay.js`; live and replay
  paths render identically.

### Tunables (App.tunables)

- HMM: `muH, muL, sigma, pHH, pLL`
- Population: `N` (6–200), `risk/cognitive/reaction` 3-slider mixes
  (sum=100), `biasShare`, `llmShare`, `tracedCount`
- Agent mechanics: `biasAmount, valuationNoise, passiveFillProb,
  priorMix, discountGamma, panicAskBias, trustLambda, applyBias,
  applyNoise, peerWindow`
- Regulator: `regulatorMode ∈ {off, warn, halt}, regulatorThreshold,
  regulatorPersistence, regulatorCooldown, regulatorHaltTicks,
  regulatorDiscount`
- Run: `speed` (ticks/sec)

### Regulator semantics

1. Engine calls `regulator.observe({tick, lastPrice, refEstimator, ...})`
   once per tick after price recording.
2. If `ρ = P / refFV ≥ threshold` for `persistence` consecutive ticks,
   regulator posts a `regulator-warning` message to the bus (`toId:
   'all'`) and enters a `cooldown` period during which no new warning
   fires. In `halt` mode, trading is frozen for `haltTicks` afterwards.
3. `regulator.isActive(tick)` returns true for `discountTicks` following
   the last warning — agents with `regulatorReaction = 'discount'`
   scale V̂ by `(1 − γ)`; `panic` agents scale by `(1 − 2γ)` and shift
   their passive ask below V̂ by `panicAskBias`.

### LLM scaffold

`js/llm.js` exposes `LLM.buildPrompt(agent, market, V, tick, ctx)` and
`LLM.complete(prompt, ...)`. The stub is deterministic; replace the
heuristic branch inside `complete()` with an async Anthropic/OpenAI
call when wiring a real backend. The prompt already contains a
`MARKET ALERTS:` section fed by `regulator-warning` messages, so the
hookup point is just the HTTP call + an async-aware engine loop.

## Verifying changes

- No build, no tests, no package manager. Open `index.html` directly.
- Smoke test: Start → agents populate the table, price chart begins to
  trace, bubble ratio oscillates around 1. Crank `biasShare` up to
  trigger regulator warnings.
- Watch the browser console — agents and engine throw on invariant
  violations rather than logging warnings.
- Export JSON via the ⬇ button to inspect a full session (events,
  aggregates, per-round final cash, traced-agent counts).

## Working in this repo

- Prefer editing existing modules over adding new ones — the module
  boundaries above are load-bearing for the replay system.
- Keep the code framework-free and dependency-free. All JS files use
  `'use strict'` and are loaded as plain `<script>` tags from
  `index.html`. Load order in `index.html` matters (dependencies before
  consumers); see the current list there as authoritative.
- Match the existing commenting style: module header block explaining
  the role of the file, short inline comments only where the *why* is
  non-obvious.
- Live site is served via GitHub Pages (`CNAME` in repo root, deploy
  workflow under `.github/workflows/`). `latex/` and `pdf/` are paper
  artifacts, not part of the runtime.
