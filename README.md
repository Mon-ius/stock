# Virtual Trading Plus — Counterfactual

A fully self-contained, browser-based continuous double auction for studying
**bubbles under bounded rationality with a regulator intervention**. The
three-plan structure from the original replication is preserved —

- **Plan I · algorithmic.** Expected-utility argmax agents (`UtilityAgent`).
- **Plan II · LLM + forms.** Prompt carries full risk-preference forms
  `{U_L, U_N, U_A}`, belief, cognitive tier, and regulator alerts.
- **Plan III · LLM + label.** Prompt carries only the risk label.

— but built on top of a new foundation:

1. **Large population.** The market scales from N = 6 to **N ≤ 200** traders,
   rendered through aggregated fan charts + an opt-in per-agent trace log
   (not N-line overplots).
2. **New asset type.** The per-period dividend is no longer the deterministic
   `{0, 2·μ}` Bernoulli draw. It is generated by a **two-state hidden Markov
   model** `(μ_H, μ_L, σ, p_HH, p_LL)` with regime-conditional Gaussian
   emissions. The fundamental value is computable analytically, but not
   trivially — agents with weak cognitive capacity must filter for the hidden
   regime from observed dividends.
3. **Regulator.** A regulator observes the rolling price-to-FV ratio
   `ρ_t = P_t / FV_t` and, when `ρ` sustains above a threshold for a
   configurable persistence window, posts a `regulator-warning` message on the
   agent bus. Agents react by their `regulatorReaction` profile —
   `ignore`, `discount V̂`, or `panic` (dump inventory). A `halt` mode freezes
   the book for a set number of ticks following a warning.

Pure HTML, CSS, and vanilla JavaScript. No frameworks, no build step, no
external libraries. Open `index.html` in any modern browser to run.

## What it does

- Simulates a continuous double auction where a finite-life asset is traded
  over **T periods** (default 10, configurable). The per-period dividend is
  drawn by a two-state HMM; at the start of period *t* the analytical FV is
  ```
  FV_t = Σ_{k=1..T−t+1} E[d_{t+k−1} | belief_t]
  ```
  where `belief_t` is the regulator's forward Bayesian posterior over the
  hidden regime, updated each tick from the realised dividend.
- Every trader is an orthogonal product across four axes: **risk preference**
  (`loving U=w²` / `neutral U=w` / `averse U=√w`), **cognitive capacity**
  (`naive sample-mean` / `Kalman windowed filter` / `analytical exact HMM`),
  **regulator reaction** (`ignore` / `discount` / `panic`), and an
  optional **bias flag** + **LLM flag**. Population mix is controlled by three
  sum-to-100 slider triplets plus a bias share + LLM share.
- The cognitive tier determines which FV estimator an agent uses to form its
  subjective valuation `V̂_{i,t}`. Analytical agents see the true FV; Kalman
  agents run a windowed filter with a biased prior; naive agents use a sample
  mean. Prior bias, prior noise, and peer trust blending are all stacked on
  top inside `UtilityAgent.decide()`.
- The regulator is wired into the same message bus the agents use to broadcast
  `valuation-report` and `peer-broadcast` messages. LLM agents receive
  regulator alerts via a `MARKET ALERTS:` section in the prompt.

## Features

- **Price-time priority order book** with self-match prevention.
- **Seedable PRNG** (mulberry32). Every run is reproducible from its
  `(seed, tunables)` pair. Two independent streams — one for the engine, one
  for sampling — so endowment edits + rebuild reproduce the matched run.
- **Sortable agent table** with click-to-expand per-row detail. Replaces
  per-agent cards. Safe at N=200.
- **Per-agent opt-in tracing.** Toggle the checkbox on any row to start
  recording the full decision trail for that agent; everyone else is rolled
  into per-tick cohort quantiles in the aggregate store (O(ticks) instead of
  O(N·ticks)).
- **Aggregated fan charts.** Cohort valuation fan (IQR + envelope) with
  traced-agent overlays, instead of overplotting N lines.
- **N×N trust matrix** backed by a single `Float32Array(N·N)`, rendered as
  a canvas blit. O(1) `get/set`, `O(k log k)` `topK`, 40 KB at N=100.
- **Regulator timeline** panel showing ρ, threshold, and warning events.
- **Universal Parameters panel.** HMM `(μ_H, μ_L, σ, p_HH, p_LL)`, population
  size, risk / cognitive / reaction mixes, bias + LLM share, traced count,
  regulator mode / threshold / persistence / cooldown / discount span / γ,
  belief adjustments, run speed.
- **Export session to JSON** with full events, aggregates, per-round final
  cash, traced-agent decision trails, and the message log.
- **Dark / light / auto theme** with canvas colors synced from CSS custom
  properties.

## Regulator semantics

1. Engine calls `regulator.observe({tick, lastPrice, refEstimator, ...})`
   once per tick after price recording.
2. If `ρ = P / refFV ≥ threshold` for `persistence` consecutive ticks, the
   regulator posts a `regulator-warning` message to the bus (`toId: 'all'`)
   and enters a `cooldown` period during which no new warning fires. In
   `halt` mode, trading is frozen for `haltTicks` afterwards.
3. `regulator.isActive(tick)` returns true for `discountTicks` following the
   last warning — agents with `regulatorReaction = 'discount'` scale V̂ by
   `(1 − γ)`; `panic` agents scale by `(1 − 2γ)` and shift their passive ask
   below V̂ by `panicAskBias`.

## LLM scaffold (Plan II / III)

`js/llm.js` exposes `LLM.buildPrompt(agent, market, V, tick, ctx)` and
`LLM.complete(prompt, ...)`. The stub is deterministic; the hookup point
for a real Claude / OpenAI call is marked inside `complete()`. The prompt
already contains a `MARKET ALERTS:` section fed by `regulator-warning`
messages, so the remaining work is just the HTTP call + an async-aware
engine loop.
